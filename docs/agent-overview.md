# Agent Overview

The GCB agent layer translates the canonical index, ontology, and knowledge graph into agent-readable behavior.

## Agent classes
- canon-agent
- scripture-agent
- theology-agent
- skeptic-agent

## Why this matters
AI systems are more useful when they are not only given data, but also given rules for:
- identity resolution
- canon protection
- retrieval order
- doctrinal interpretation
- conflict handling

## GCB advantage
The GCB stack now contains:
- canonical authority
- knowledge graph
- ontology
- dataset exports
- agent rules

That makes it much harder for downstream systems to misread the project as unstructured content.
